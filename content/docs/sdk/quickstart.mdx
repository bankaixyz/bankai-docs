This is the shortest path to a working proof flow. You configure the
SDK, ask for the on-chain data you care about, and get back a verified
result you can trust.

## The mental model

The SDK ships as two main crates. `bankai-sdk` handles querying the
Bankai API and constructing MPT proofs for arbitrary on-chain data.
`bankai-verify` takes the resulting proof bundle and verifies it
automatically -- the verifier is also zkVM-compatible, so you can
run it inside a proving system.

Most users should start with the batch builder in `bankai-sdk`, not
the low-level API client.

## Installation

Add the crates from the
[bankai-sdk](https://github.com/bankaixyz/bankai-sdk) repository.
Pin to the latest release tag:

```toml
[dependencies]
bankai-sdk = { git = "https://github.com/bankaixyz/bankai-sdk", tag = "v0.1.2.1" }
bankai-verify = { git = "https://github.com/bankaixyz/bankai-sdk", tag = "v0.1.2.1" }
bankai-types = { git = "https://github.com/bankaixyz/bankai-sdk", tag = "v0.1.2.1" }
```

You also need the patched `ethereum_hashing` crate:

```toml
[dependencies]
ethereum_hashing = { git = "https://github.com/bankaixyz/ethereum_hashing", rev = "c457c3e", default-features = false, features = ["portable"] }

[patch.crates-io]
ethereum_hashing = { git = "https://github.com/bankaixyz/ethereum_hashing", rev = "c457c3e" }
```

## What you need

Depending on what you want to verify, you may need:

- an Ethereum execution RPC
- an Ethereum beacon RPC
- a map of OP Stack RPCs keyed by chain name, such as `"base"` or
  `"worldchain"`

You don't need to configure every RPC up front. Only provide the ones
needed for the proofs you plan to request.

## Configure the SDK

Start with `Network::Sepolia`:

```rust
use bankai_sdk::{Bankai, Network};

let bankai = Bankai::new(
    Network::Sepolia,
    Some("https://sepolia.infura.io/v3/YOUR_KEY".to_string()),
    Some("https://sepolia.beacon-api.example.com".to_string()),
    None,
);
```

Pass only the RPCs you need. `Network::Local` and custom API base
URLs are mainly for local Bankai deployments -- they aren't the main
onboarding path.

## First proof flow

Smallest useful end-to-end example:

```rust
use alloy_primitives::Address;
use bankai_sdk::{Bankai, HashingFunction, Network};
use bankai_verify::verify_batch_proof;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let bankai = Bankai::new(
        Network::Sepolia,
        Some("https://sepolia.infura.io/v3/YOUR_KEY".to_string()),
        Some("https://sepolia.beacon-api.example.com".to_string()),
        None,
    );

    let proof_bundle = bankai
        .init_batch(Network::Sepolia, None, HashingFunction::Keccak)
        .await?
        .ethereum_execution_header(9_231_247)
        .ethereum_account(9_231_247, Address::ZERO)
        .execute()
        .await?;

    let results = verify_batch_proof(proof_bundle)?;

    println!("Verified block {}", results.evm.execution_header[0].number);
    println!("Verified balance {}", results.evm.account[0].balance);

    Ok(())
}
```

## What happens here

- `init_batch(...)` anchors the request to a Bankai block
- the batch builder collects the exact proof requests you need
- `.execute()` fetches one optimized `ProofBundle`
- `verify_batch_proof(...)` checks the whole trust chain and returns
  verified outputs
